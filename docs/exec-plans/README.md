# Execution Plans

Use checked-in plans for multi-step or high-risk work.

## Active Plans

Create `active/` with the first plan. Use this name pattern:

```text
YYYY-MM-DD-short-slug.md
```

## Completed Plans

Create `completed/` and move finished plans there after validation.

## Required Sections

- Goal.
- Acceptance criteria.
- Context links.
- Steps.
- Validation.
- Decision log.
- Progress log.
